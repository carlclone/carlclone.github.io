接口安全和 WEB 安全

 

收集自工作经历 , OWASP TOP10 和 Pitfall , 更多可参考 CWE Software Development Security 列表

1 使用 HTTPS
有免费证书
性能也不再是问题

2 可预测的 ID 枚举
如果没有严格的认证检查 , 危险
没有验证用户数据权限

检查用户是否该数据的拥有者

使用不可预测的 UUID 但不要用作数据库主键

3 session 攻击主要在猜和偷 session id

4 client 端实现 session 被认为是不安全的, 很容易获取到
需要进行完整性校验
可选的加密机制来防止信息暴露
机密性和完整性

5 JWT RFC7519

JWT 是 base64 编码的, 没有提供任何保护
完整性由签名保证, 机密性confidentiality 由加密保证

6 如果安全域过大 , 请用非对称加密 , 在自己的安全域里可以使用对称加密 (指签名)

7 cookie
