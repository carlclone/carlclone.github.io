使用 symfony 组件构建自己的 framework

  • DRAFTS_3
  • FRAMEWORK 相关
  • BUILD-YOUR-OWN-FRAMEWORK

 

一个 web 框架的重点不是 MVC,而是关注分离

一个 web 框架是围绕 http 构建的,http 请求的 3 个关键要素就是输入,处理请求,输出

使用框架的目的,安全性和可测试性

PHP 自带的请求参数有$_GET 等等,改进代码的第一步是使用 symfony 的 http 基础库将请求和响应对象化

使用 smyfony 库的一个原因,代码经过严格的安全审计
关于判断客户端 IP 获取的安全性问题
REMOTE_ADDR 和HTTP_X_FORWARDED_FOR

如果没有代理服务器,那么客户端可以很容易地伪造 header 并欺骗服务端程序
而 symfony 的 getClientIP 需要显式地设置信任的代理服务器 IP,因此不会有这个安全问题
,这又是一个使用框架的理由

第二步,当你为你的站点添加更多页面时,读取请求和发送响应的代码就发生了重复,
这时候就需要进一步的抽离封装代码,将页面的逻辑放到另一个文件中,通过 require 引入

第三步,为了增加灵活性和安全性(避免文件和 url 的直接映射) , 将分发的逻辑放到 web 服务器直接完成,这里就引入了前端控制器
,相当于一个只带完全匹配功能的路由,并且将代码放到其他目录,对外只暴露前端控制器代码

第四部,页面的逻辑这时候可以转换成 PHP 模板,用 ob_start()和 ob_get_clean()包围起来

第五步,使用 extract 将参数变成基本的变量,而不是数组,方便模板填数据

第六步,使用 symfony 的路由组件,让参数成为 url 的一部分,放入 app.php 中,作为应用程序的一部分,而不是对外暴露的一部分

第七步,路由可以进一步优化,增加缓存,避免每次都重新生成路由表,这里有一个装饰者模式的CompiledUrlMatcherDumper

第八步,引入控制器,分离模板代码和逻辑,形式可以是闭包,函数,类

使用 httpkernel 延迟加载 controller 类 , kernel 还通过反射注入正确的参数到方法中

提供可重用性,可测试性,减少 front.php 的代码
