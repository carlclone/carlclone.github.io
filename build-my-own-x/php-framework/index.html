构建自己的 PHP 框架

从最原始的 echo 出发 , 衍生出一套框架的过程

使用 symfony/http-foundation 库的理由, 保证 HTTP 规范, 经过代码审查和大量应用, 安全性更高, 请求和响应对象化

使用单一入口的理由, 减少代码文件暴露到外网的风险

使用 symfony/routing 的理由, 路由美化, 匹配规则更灵活

使用 http-kernel 的理由, 延迟解析控制器, 自动注入参数

将请求处理逻辑移到 Framework 类的理由, 关注分离原则, 保持对外暴露的 front.php 干净, 只负责请求框架的实例化和 handle()方法调用

引入 phpunit, 并将注入的参数改为接口类型, 提高可测试性
